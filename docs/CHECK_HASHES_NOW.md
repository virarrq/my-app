# Проверка хешей - Быстрая инструкция

## Что вы увидели в базе данных - это ПРАВИЛЬНО! ✅

Хеши **НЕ хранятся** в базе данных. Они вычисляются динамически при отправке ответа.

## Как проверить прямо сейчас:

### 1. Откройте приложение в браузере

```
http://127.0.0.1:5500/client/dashboard.html
```

### 2. Откройте Console (F12 → Console)

### 3. Обновите страницу или загрузите задачи

Вы должны увидеть в консоли:

```
📊 Проверка хешей задач:
  test:
    Checksum ✅: 3a37b0a1
    Hash ✅: f7aa4f01dc164ad0...
```

### 4. Если видите ✅ - все работает!

Если видите ❌ - хеши не приходят, нужно проверить сервер.

## Проверка через Network

1. F12 → Network
2. Обновите страницу
3. Найдите запрос `tasks`
4. Кликните на него
5. Вкладка **Response**

Должны увидеть:

```json
[
  {
    "id": "...",
    "title": "test",
    "checksum": "3a37b0a1",
    "hash": "f7aa4f01dc164ad02d1198a856712954bccc65c5167aedb51c5a320515684fcd"
  }
]
```

## Если хешей нет:

1. Убедитесь, что сервер запущен:
   ```bash
   cd server
   npm start
   ```

2. Проверьте, что нет ошибок в консоли сервера

3. Перезапустите сервер (Ctrl+C, затем npm start)

## После проверки

Можете удалить временный console.log из `client/js/api.js` (строки с "Проверка хешей").

---

**Важно:** Хеши в базе данных НЕ нужны! Это правильная архитектура.
