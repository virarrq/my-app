# Планировщик задач (Task Planner)

Веб-приложение для управления задачами с календарём, фильтрами и уведомлениями.

## Описание проекта

Полнофункциональное приложение для планирования задач с аутентификацией пользователей, возможностью создания, редактирования и удаления задач, а также с календарным представлением, кастомными пикерами даты/времени и системой уведомлений.

## Технологии

### Frontend
- Чистый JavaScript (ES6+)
- HTML5 / CSS3
- Supabase Client (для аутентификации)

### Backend
- Node.js + Express
- Supabase (база данных PostgreSQL + аутентификация)
- JWT токены для авторизации

### База данных
- PostgreSQL (через Supabase)
- Row Level Security (RLS) для защиты данных

## Основные функции

### 1. Аутентификация (`client/js/auth.js`)

#### `checkAuth()`
Проверяет наличие активной сессии пользователя.

#### `requireAuth()`
Требует авторизацию — перенаправляет на страницу входа если пользователь не авторизован.

#### `getAccessToken()`
Получает JWT токен текущей сессии для API запросов.

#### `login(email, password)`
Авторизует пользователя по email и паролю.

#### `register(email, password)`
Регистрирует нового пользователя.

#### `logout()`
Выход из системы и очистка сессии.

### 2. API запросы (`client/js/api.js`)

#### `apiCall(endpoint, options)`
Универсальная функция для отправки запросов к API с автоматическим добавлением JWT токена.

#### `taskAPI.getAll(filters)`
Получает все задачи пользователя с возможностью фильтрации по:
- дате
- приоритету (low, medium, high)
- статусу (pending, completed)
- поисковому запросу

#### `taskAPI.create(taskData)`
Создаёт новую задачу.

#### `taskAPI.update(id, taskData)`
Обновляет существующую задачу.

#### `taskAPI.delete(id)`
Удаляет задачу.

#### `taskAPI.updateStatus(id, status)`
Изменяет только статус задачи (pending/completed).

### 3. Дашборд (`client/js/dashboard.js`)

#### `init()`
Инициализирует приложение — проверяет авторизацию, загружает задачи, настраивает интерфейс.

#### `setupUI()`
Настраивает пользовательский интерфейс — отображает email, имя, аватар пользователя.

#### `setupEventListeners()`
Подключает все обработчики событий для кнопок, форм, фильтров. Инициализирует кастомные пикеры даты и времени.

#### `makeDatePicker(config)`
Универсальная фабрика кастомного пикера даты. Создаёт попап-календарь с навигацией по месяцам, подсветкой сегодняшней и выбранной даты. Автоматически корректирует позицию если попап выходит за границы экрана. Используется для поля даты в форме задачи и для фильтрации.

#### `makeTimePicker(config)`
Универсальная фабрика кастомного пикера времени. Реализует колёсный скролл (scroll-wheel) с двумя столбцами — часы (00–23) и минуты (00–59). Выбранное значение подсвечивается полосой по центру. Применяет JS-снэппинг для точного выравнивания.

#### `loadTasks(filters)`
Загружает задачи с сервера с применением фильтров.

#### `applyViewFilter()`
Применяет фильтр текущего вида (все, сегодня, предстоящие) и перерисовывает задачи. Использует локальное время (без UTC-смещения).

#### `handleViewChange(view)`
Переключает между видами: все задачи, сегодня, предстоящие, календарь.

#### `openTaskModal(task)`
Открывает модальное окно для создания или редактирования задачи. Устанавливает значения кастомных пикеров даты и времени.

#### `handleTaskSubmit(e)`
Обрабатывает отправку формы задачи (создание/обновление). Включает клиентскую валидацию даты.

#### `handleToggleStatus(id, currentStatus)`
Переключает статус задачи между pending и completed.

#### `handleDeleteTask(id)`
Удаляет задачу после подтверждения.

#### `openProfileModal()`
Открывает модальное окно профиля пользователя.

#### `updateDisplayName(e)`
Обновляет отображаемое имя пользователя.

#### `uploadAvatar(e)`
Загружает аватар пользователя в Supabase Storage.

#### `checkUpcomingTasks()`
Проверяет задачи и отправляет уведомления за 10 минут до начала.

#### `checkOverdueTasks()`
Каждые 30 секунд проверяет изменение количества просроченных задач. При изменении автоматически перерисовывает карточки без перезагрузки страницы — просроченные задачи немедленно получают статус «ПРОСРОЧЕНО».

### 4. Интерфейс (`client/js/ui.js`)

#### `renderTasks(tasks)`
Отрисовывает список задач в виде карточек.

#### `createTaskCard(task)`
Создаёт HTML разметку для одной карточки задачи. Вычисляет статус просрочки через `checkIfOverdue()`.

#### `checkIfOverdue(task)`
Сравнивает дату/время задачи с текущим локальным временем. Возвращает `true` если задача не выполнена и дедлайн прошёл.

#### `addTaskEventListeners(container)`
Добавляет обработчики событий к карточкам задач (чекбоксы, кнопки редактирования/удаления).

#### `renderCalendar(date, tasks, onDateClick)`
Отрисовывает календарь с отметками дней, в которые есть задачи. Подсвечивает сегодняшний день по локальному времени.

### 5. Интернационализация (`client/js/i18n.js`)

#### `t(key)`
Возвращает перевод для указанного ключа на текущем языке.

#### `setLanguage(lang)`
Устанавливает язык интерфейса (en/ru) и обновляет все тексты.

#### `getCurrentLanguage()`
Возвращает текущий язык интерфейса.

#### `updateUI()`
Обновляет все элементы интерфейса с переводами.

### 6. Утилиты (`client/js/utils.js`)

#### `getLocalDateStr(date?)`
Возвращает дату в формате `YYYY-MM-DD` по **локальному** времени без UTC-конвертации. Используется везде вместо `new Date().toISOString().split('T')[0]` для корректной работы в любом часовом поясе.

#### `debounce(func, wait)`
Задерживает выполнение функции — используется для оптимизации поиска.

#### `escapeHtml(text)`
Защищает от XSS атак — экранирует HTML символы.

#### `formatDate(dateStr)`
Форматирует дату — показывает «Сегодня»/«Today», «Завтра»/«Tomorrow» или локализованную дату. Сравнение выполняется через локальные компоненты даты (без UTC-конвертации).

#### `formatTime(timeStr)`
Форматирует время в формат HH:MM.

#### `getUserInitials(email)`
Получает первую букву email для аватара по умолчанию.

## Серверная часть

### Middleware (`server/middleware/auth.js`)

#### `authMiddleware(req, res, next)`
Проверяет JWT токен в заголовке Authorization и добавляет пользователя в req.user.

### API маршруты (`server/routes/tasks.js`)

#### `GET /api/tasks`
Получает все задачи пользователя с фильтрацией.

Параметры запроса:
- `date` — фильтр по дате
- `priority` — фильтр по приоритету
- `status` — фильтр по статусу
- `search` — поиск по названию и описанию

#### `POST /api/tasks`
Создаёт новую задачу.

Тело запроса:
```json
{
  "title": "Название задачи",
  "description": "Описание",
  "date": "2024-12-31",
  "time": "14:30",
  "priority": "high",
  "status": "pending"
}
```

#### `PUT /api/tasks/:id`
Обновляет задачу полностью.

#### `PATCH /api/tasks/:id/status`
Обновляет только статус задачи.

Тело запроса:
```json
{
  "status": "completed"
}
```

#### `DELETE /api/tasks/:id`
Удаляет задачу.

#### `validateTask(task)`
Проверяет корректность данных задачи перед сохранением.

### Сервер (`server/server.js`)

Главный файл сервера Express с настройкой middleware, маршрутов и обработкой ошибок.

## База данных (`database/schema.sql`)

### Таблица `tasks`

Поля:
- `id` — UUID, первичный ключ
- `user_id` — UUID, ссылка на пользователя
- `title` — текст, название задачи
- `description` — текст, описание (необязательно)
- `date` — дата выполнения (DATE)
- `time` — время выполнения (TIME)
- `priority` — приоритет (low, medium, high)
- `status` — статус (pending, completed)
- `created_at` — время создания

### Индексы
- По user_id для быстрого поиска задач пользователя
- По date для сортировки по датам
- По комбинации user_id + date
- По status и priority для фильтрации

### Row Level Security (RLS)
Настроены правила безопасности:
- Пользователи видят только свои задачи
- Пользователи создают задачи только для себя
- Пользователи редактируют только свои задачи
- Пользователи удаляют только свои задачи

## Установка и запуск

### 1. Настройка базы данных

Создайте проект в Supabase и выполните SQL из `database/schema.sql`.

### 2. Настройка сервера

```bash
cd server
npm install
```

Создайте файл `.env`:
```
SUPABASE_URL=ваш_url
SUPABASE_SERVICE_ROLE_KEY=ваш_ключ
PORT=3000
```

Запуск:
```bash
npm start
```

### 3. Настройка клиента

Отредактируйте `client/js/config.js`:
```javascript
export const CONFIG = {
  SUPABASE_URL: 'ваш_url',
  SUPABASE_ANON_KEY: 'ваш_публичный_ключ',
  API_URL: 'http://localhost:3000/api'
};
```

Откройте `client/index.html` в браузере или используйте локальный сервер.
